Jesse Dashboard

Run Jesse's GUI backtester and live-trade plugin alongside our TradeCore tools.

Quick start

Bring up the dockerized stack with the helper script:

scripts/run_jesse.sh up

Stop it with scripts/run_jesse.sh down; stream logs via scripts/run_jesse.sh logs.

Default dashboard password (from Jesse/tradecore-jesse/.env): tradecore.

Ports & directories

Dashboard

http://localhost:9000

GUI for backtests + live trading

Jupyter (inside container)

http://localhost:8888

Use the token printed in logs

Project root

Jesse/tradecore-jesse

Mounted to /home inside the container

Persistent storage

Jesse/tradecore-jesse/storage

Charts, logs, exports stay in-repo

Notes

  • Services are defined in Jesse/tradecore-jesse/docker/docker-compose.yml (Postgres, Redis, and the Jesse app).
  • If you change ports or DB credentials, update both .env and the compose file so the container can reach Postgres/Redis.
  • Strategies live under Jesse/tradecore-jesse/strategies; keep new work there to keep everything contained in the Jessie folder.
